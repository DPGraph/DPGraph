Overview of Empirical Findings

There are four input variables that we can adjust when calculating the output of graph-DP algorithms. These four variables are:
  • Algorithm
  • Dataset
  • Query
  • Epsilon
It's interesting to investigate the effect of each of these variables on the accuracy of output. More specifically, fixing three of the four variables, we may be interested in whether or not the fourth variable affects the results, and if yes, how does it do so. We aim to address these questions from the empirical findings of our experiments. To see more details for each variable, click on the corresponding titles.

Algorithm:

Q: Is there one algorithm that outperforms the rest, across various input settings?

A: For degree distribution, no single algorithm dominates all others. For subgraph counting, Ladder is the dominant algorithm.

More details

For degree distribution Node-DP algorithms, edgeAdd_degHisPart_Lap, edgeAdd_degCum_Lap, and edgeAdd_degCum_Lap_variant consistently outperforms degHis_Lap, degSeq_Lap, and nodeTrun_Smooth, across all input settings.

For smaller values of e, edgeAdd_degHisPart_Lap is the best performing algorithm.

For larger values of e, edgeAdd_degCum_Lap is the best performing algorithm.

For degree distribution Edge-DP algorithms, degHis_Lap outperforms degSeq_Lap.

For subgraph counting algorithms, Ladder outperforms Laplace and Smooth across all input settings. This is consistent with findigns in the paper. [?]


Dataset:

Q: Do graph-DP algorithms’ performances vary on graph datasets with different shapes?

A: Yes, graph datasets' shapes have an impact on algorithms' performances.

More details

For subgraph counting queries, Laplace algorithm's error increases with the number of nodes in a graph dataset. This is because Laplace algorithm achieves differential privacy by adding noise proportional to the global sensitivity of a dataset. And the global sensitiviy of a graph dataset increases with the number of nodes.

For Ladder and Smooth algorithm, graph datasets' shape have an impact on their performances. But exact which properties of the graph dataset have the most impact on the algorithms' performances remain to be answered.


Query:

Q: (For SGC queries) Do graph-DP algorithms’ performances vary on different queries?

A: Yes, queries have an impact on graph-DP algorithms' performances.

More details

For subgraph counting queries, some queries have more accurate results than others, fixing all other input variables.

Here is the observed ranking of errors for the subgraph counting queries:

Err(Triangle) & Err(3-kstar) < Err(2-ktriangle) < Err(4-kclique)

Intuitively, 4-kclique and 2-ktriangle are the "harder" queries to answer. On the other hand, there is no clear ordering of the errors for triangle counting and 3-kstar counting. One query has more accurate results than the other for some combinations of algorithm and dataset, and vise versa.


Epsilon:

Q: Do graph-DP algorithms' performances vary on different epsilon values?

A: Yes. As we expected, algorithms' performances increases as epsilon increases.

More details

Intuitively, a larger epsilon corresponds to a weaker level of privacy guarantee. Thus, less noise is needed, which leads to higher accuracy of output.

This behavior is observed across all input settings. To take a look at epsilon's effect, set all the other variables.
